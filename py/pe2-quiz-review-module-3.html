Question 1: LIFO Data Structure

A data structure described as **LIFO** is actually a:

Explanation

LIFO stands for **Last-In, First-Out**, which is the fundamental principle governing the behavior of a **stack** data structure. The element added last is the first one to be removed.

Question 2: Valid Python Class Instantiation

If the class's constructor is declared as below, which one of the assignments is valid? 

1 class Class:
2     def __init__(self):
3         pass
Explanation

To create an instance of a class, you call the class name followed by parentheses: ClassName(). The self parameter is passed **automatically** by Python; since the __init__ method takes no other arguments, calling Class() is the correct way.

Analysis of Other Options
  • (A, C): These attempt to pass an extra argument, which would raise a TypeError.
  • (D): This assigns the **class object itself** (the blueprint), not an instance of the class.

Question 3: Calling Superclass Constructor

If there is a superclass named A and a subclass named B, which one of the presented invocations should you put instead of the comment? 

1 class A:
2     def __init__(self):
3         self.a = 1
4
5 class B(A):
6     def __init__(self):
7         # Put selected line here.
8         self.b = 2
Explanation

When calling the superclass method **directly** using the class name (e.g., A.__init__), you must explicitly pass the instance to be initialized as the first argument, which is **self**. (The preferred, modern way is super().__init__().)

Question 4: Name Mangling and Access

What will be the effect of running the following code? 

1 class A:
2     def __init__(self,v):
3         self.__a = v + 1
4
5 a = A(0)
6 print(a.__a)
Explanation

The attribute self.__a (with two leading underscores) is subject to **name mangling**. It is renamed internally to _A__a. When line 6 attempts to access it using the original name a.__a, the attribute is not found, resulting in an **AttributeError**.

Question 5: Method Execution and Return Value

What will be the output of the following code? 

 1 class A:
 2     def __init__(self,v = 1):
 3         self.v = v
 4
 5     def set(self,v):
 6         self.v = v
 7         return v
 8
 9 a = A()
10 print(a.set(a.v + 1))
Explanation

1. **Initialization:** a = A() sets a.v to **1** (default value).

2. **Argument Evaluation:** The expression a.v + 1 evaluates to $1 + 1 = \mathbf{2}$.

3. **Method Call:** a.set(2) is called. Inside set, self.v is updated to 2, and the method **returns 2**.

4. **Output:** print() prints the **return value** of the a.set() call, which is **2**.

Question 6: Class vs. Instance Variables

What will be the output of the following code? 

 1 class A:
 2     X = 0
 3     def __init__(self,v = 0):
 4         self.Y = v
 5         A.X += v
 6
 7 a = A()
 8 b = A(1)
 9 c = A(2)
10 print(c.X)
Explanation

X is a **class variable** (shared by all instances) and is modified directly using A.X in the constructor. Its value accumulates with each instantiation:

  • Initial A.X: 0
  • a = A() (v=0): $0 + 0 = \mathbf{0}$
  • b = A(1) (v=1): $0 + 1 = \mathbf{1}$
  • c = A(2) (v=2): $1 + 2 = \mathbf{3}$

print(c.X) retrieves the final class variable value, **3**.

Question 7: hasattr Function

What will be the output of the following code? 

1 class A:
2     A = 1
3
4 print(hasattr(A, 'A'))
Explanation

hasattr(A, 'A') checks if the class object A possesses an attribute named 'A'. Since A = 1 on Line 2 defines a class variable named **A**, the check returns **True**.

Question 8: Instantiation Error

What will be the result of executing the following code? 

1 class A:
2     def __init__(self):
3         pass
4
5 a = A(1)
6 print(hasattr(a, 'A'))
Explanation

The code on Line 5, a = A(1), attempts to pass an argument to the constructor. However, A.__init__(self) is defined to accept only the implicit self argument. This raises a **TypeError** exception before the print statement is ever reached.

Question 9: Inheritance and __str__ MRO

What will be the result of executing the following code? 

 1 class A:
 2     def __str__(self):
 3         return 'a'
 4
 5 class B(A):
 6     def __str__(self):
 7         return 'b'
 8
 9 class C(B):
10     pass
11
12 o = C()
13 print(o)
Explanation

print(o) calls o.__str__(). The Method Resolution Order (MRO) is **C** → **B** → **A**. Python finds __str__ in the closest parent, class **B**, and executes it, which returns **'b'**. This is called method overriding.

Question 10: issubclass Transitivity

What will be the result of executing the following code? 

 1 class A:
 2     pass
 3
 4 class B(A):
 5     pass
 6
 7 class C(B):
 8     pass
 9
10 print(issubclass(C, A))
Explanation

Inheritance is **transitive**: since C inherits from B, and B inherits from A, class **C is indirectly a subclass of A**. The issubclass() function checks for both direct and indirect inheritance, so it returns **True**.

Question 11: Multiple Inheritance Method Resolution

What will be the result of executing the following code? 

 1 class A:
 2     def a(self):
 3         print('a', end='')
 4
 5 class B:
 6     def a(self):
 7         print('b', end='')
 8
 9 class C(B, A):
10     def c(self):
11         self.a()
12
13 o = C()
14 o.c()
Explanation

The class definition is class C(B, A). When self.a() is called from within C.c(), the Method Resolution Order (MRO) is **C** → **B** → **A**. The method a() is found first in class **B**, due to its priority in the inheritance list, and executes, which prints **'b'**.

Question 12: Multiple Inheritance __str__ MRO

What will be the result of executing the following code?

 3         return 'a'
 4
 5 class B:
 6     def __str__(self):
 7         return 'b'
 8
 9 class C(A, B):

The class definition is class C(A, B). The MRO is **C** → **A** → **B**. Python finds __str__ in the first parent specified in the inheritance list, class **A**, and executes it, which returns **'a'**.

Question 13: Inheritance and Attribute Shadowing

What will be the result of executing the following code? 

 1 class A:
 2     v = 2
 3
 4 class B(A):
 5     v = 1
 6
 7 class C(B):
 8     pass
 9
10 o = C()
11 print(o.v)
Explanation

The attribute lookup for o.v follows the MRO: Instance → C → **B** → A. The class variable v is found in class **B** where it is set to **1**. This value shadows (overrides) the value 2 set in class A.

Question 14: try...except...else...finally Flow

What will be the result of executing the following code? 

 1 def f(x):
 2     try:
 3         x = x / x
 4     except:
 5         print("a",end='')
 6     else:
 7         print("b",end='')
 8     finally:
 9         print("c",end='')
10
11 f(1)
12 f(0)
Explanation

1. **f(1)** (Success): Executes try, skips except, runs else (b), runs finally (c). Output: **bc**.

2. **f(0)** (Failure): Runs try (raises ZeroDivisionError), runs except (a), skips else, runs finally (c). Output: **ac**.

Combined output: **bcac**.

Question 15: Exception Arguments

What will be the result of executing the following code? 

1 try:
2     raise Exception(1,2,3)
3 except Exception as e:
4     print(len(e.args))
Explanation

Arguments passed to an exception constructor (e.g., 1, 2, 3) are stored as a tuple in the exception object's **args** attribute. e.args is (1, 2, 3), and its length is **3**.

Question 16: Custom Exception __init__ Behavior

What will be the result of executing the following code? 

 1 class Ex(Exception):
 2     def __init__(self, msg):
 3         Exception.__init__(self, msg + msg)
 4         self.args = (msg,)
 5
 6 try:
 7     raise Ex('ex')
 8 except Ex as e:
 9     print(e)
Explanation

1. **Line 3:** Exception.__init__(self, 'exex') initializes the base class state with the concatenated string.

2. **Line 4 (The Overwrite):** self.args = (msg,) explicitly **overwrites** the instance's args attribute, setting it to the single-element tuple ('ex',).

3. **Line 9 (Print):** When print(e) is called, the exception's __str__ method uses the final value of the self.args attribute. Since self.args is a single-element tuple, ('ex',), Python prints only the element itself, resulting in **ex**.

Question 17: Implementing the Iterator Protocol

What will be the result of executing the following code? 

 1 class I:
 2     def __init__(self):
 3         self.s = 'abc'
 4         self.i = 0
 5
 6     def __iter__(self):
 7         return self
 8
 9     def __next__(self):
10         if self.i == len(self.s):
11             raise StopIteration
12         v = self.s[self.i]
13         self.i += 1
14         return v
15
16 for x in I():
17     print(x,end='')
Explanation

The class correctly implements the iterator protocol. The for loop iterates through the characters 'a', 'b', and 'c' by repeatedly calling __next__. The StopIteration exception is used to signal the end of the sequence and is handled gracefully by the for loop. The output is printed on a single line: **abc**.
