Question 1: Module Importing Syntax

Knowing that a function named fun() resides in a module named mod, choose the correct way to import it.

Explanation

The standard Python syntax to import a specific element (like a function, class, or variable) from a module directly into the current namespace is from <module_name> import <item_name>. This allows the function to be called simply as fun().

Analysis of Other Options
  • import fun from mod: **Incorrect** keyword order.
  • import fun: Imports a module named fun.
  • from fun import mod: Incorrectly reverses the module and function names.

Confirmation: The required syntax for importing a specific element (fun) from a module (mod) is from mod import fun.

Question 2: Invoking Imported Functions

Knowing that a function named fun() resides in a module named mod, and it has been imported using the following line: import mod. Choose the way it can be invoked in your code.

Explanation

When you use import mod, you bring the entire module object into the namespace. To access any item within that module, you must use the **dot notation**: module_name.item_name(), which resolves to mod.fun().

Analysis of Other Options
  • fun(): Only works if imported using from mod import fun.
  • mod->fun(): **Incorrect** syntax (uses C++/PHP arrow operator).
  • mod::fun(): **Incorrect** syntax (uses C++ scope resolution operator).

Confirmation: The standard Python convention for calling an item from an imported module object is to use mod.fun().

Question 3: Discovering Module Contents

A function which returns a list of all entities available in a module is called:

Explanation

The built-in Python function dir() is used to retrieve a list of valid attributes (functions, classes, variables, etc.) for any object, including a module. When passed a module name (e.g., dir(math)), it returns a list of all items defined within it.

Analysis of Other Options
  • listmodule(): Not a standard built-in Python function.
  • entities(): Not a standard built-in Python function.
  • content(): Not a standard built-in Python function.

Confirmation: The standard Python function for querying the contents (directory) of an object is dir().

Question 4: .pyc Files

A .pyc file contains:

Explanation

When Python runs a .py file, it first translates the source code into an intermediate format called **bytecode**. The .pyc (Python compiled) file is where this bytecode is stored, which speeds up subsequent execution by skipping the initial compilation step.

Analysis of Other Options
  • **a Python interpreter**: The interpreter is the execution program, not the file contents.
  • **a Python compiler**: The compilation happens within the interpreter.
  • **Python source code**: Source code is stored in the .py file.

Confirmation: The .pyc file stores the intermediate **compiled Python code** (bytecode).

Question 5: Module Execution

When a module is imported, its contents:

Explanation

The first time a module is imported in a Python program, its top-level code is **executed** to define its functions, variables, and initial state. Python's module loading system then caches the module, ensuring that any subsequent imports only retrieve the loaded module object and **do not re-execute** the code.

Analysis of Other Options
  • **are ignored**: Incorrect, the code must run to define the module contents.
  • **may be executed (explicitly)**: Incorrect, the execution is implicit in the act of importing.
  • **are executed as many times as they are imported**: Incorrect, they are only executed once.

Confirmation: A module's code is executed only **once** upon its first import.

Question 6: Current Module Name Variable

A predefined Python variable that stores the current module name is called:

Explanation

The special variable __name__ is assigned to every module. It stores the module's name as a string. If the file is run directly, its value is '__main__'; otherwise, it is the module's actual file name (e.g., 'my_module') when imported.

Analysis of Other Options
  • __mod__: Not a standard predefined variable for the module name.
  • __module__: Used to store the module a function or class was defined in, not the current module name at the top level.
  • __modname__: Not a standard predefined variable for the module name.

Confirmation: The predefined variable for the module name is __name__.

Question 7: Package/Module Import Hierarchy

The following statement: from a.b import c causes the import of:

Explanation

The dot notation in imports denotes hierarchy: a is the **Package**, b is the **Module** inside the package, and c is the specific **Entity** (function, class, or variable) being imported from the module.

Analysis of Other Options

Options A, B, and C incorrectly map the names to the roles of Package, Module, and Entity, reversing the correct organizational hierarchy defined by the package.module import entity structure.

Confirmation: The import hierarchy is Package.Module import Entity, making a the package, b the module, and c the entity.

Question 8: Math Module Calculation

What is the expected value assigned to the result variable after the following code is executed?

import math
result = math.e != math.pow(2, 4)
print(int(result))

Explanation
  1. math.e: Euler's number, approximately **2.71828**.
  2. math.pow(2, 4): Calculates $2^4$, which equals **16.0**.
  3. Comparison: 2.71828 != 16.0. Since the values are not equal, the expression evaluates to the Boolean value **True**.
Analysis of Other Options

The code is a Boolean comparison. The result is the Boolean value **True**. Options A and B are the operands, and Option C is the wrong Boolean result.

Confirmation: The comparison 2.71828 != 16.0 is True, which is the value assigned to the variable.

Question 9: Random Module Output

What is the expected output of the following code?

from random import randint
for i in range(2): print(randint(1, 2), end='')

Explanation

The loop executes **twice**. randint(1, 2) can only return **1 or 2**. Since the outputs are concatenated (end=''), the possible combinations are:

1st call (1) + 2nd call (1) $\rightarrow$ **11**
1st call (1) + 2nd call (2) $\rightarrow$ **12**
1st call (2) + 2nd call (1) $\rightarrow$ **21**
1st call (2) + 2nd call (2) $\rightarrow$ **22**

Analysis of Other Options

Option A is incomplete. Option B is incorrect because the set of all possible outcomes is clearly limited to four combinations. Option D is only one possible outcome.

Confirmation: The four possible outcomes are 11, 12, 21, and 22.

Question 10: Platform Module (Select two)

Choose the true statements about the platform module.

Explanation

platform.version() provides the detailed OS release version string (e.g., '10.0.19045' on Windows). platform.system() provides the generic OS name (e.g., 'Linux', 'Windows').

Analysis of Other Options
  • platform.processor() returns the **processor identity string**, not the count of running processes.
  • The third option is false; platform.python_version() returns the Python version.

Confirmation: version() and system() are correctly described.

Question 11: Bytecode Cache Directory

During the first import of a module, Python deploys the .pyc files in the directory called:

Explanation

Compiled bytecode files (.pyc) are stored in a standardized subdirectory named __pycache__, which is located inside the directory of the original .py file to speed up subsequent imports.

Analysis of Other Options
  • Hashbang: The #! line in scripts, not a directory.
  • __init__: The file that marks a directory as a Python package.
  • Mymodules: An arbitrary, non-standard directory name.

Confirmation: The standardized directory name for the bytecode cache is __pycache__.

Question 12: Shebang Digraph

The digraph written as #! is used to:

Explanation

The #! (Shebang or Hashbang) must be the first line of a script file on Unix-like operating systems (Linux/macOS). It specifies the path to the interpreter (e.g., #!/usr/bin/env python3) that the operating system should use to execute the script.

Analysis of Other Options
  • Privacy is indicated by underscores (_).
  • Docstrings are created with triple quotes (""").
  • Windows uses file associations, not the Shebang, for execution.

Confirmation: #! is the Shebang used by Unix-like operating systems.

Question 13: PIP Dependency Command

A list of package dependencies can be obtained from pip using a command named:

Explanation

The command pip show <package_name> displays detailed information about a package, including its version, location, and the **"Requires"** field, which lists its dependencies.

Analysis of Other Options
  • dir: A Python built-in function, not a pip command.
  • list: Lists all installed packages, but not their dependency details.
  • deps: Not a recognized standard pip command.

Confirmation: Detailed package information, including dependencies, is viewed with pip show.

Question 14: PIP List Command

The pip list command presents a list of:

Explanation

The pip list command queries the active Python environment and displays a list of all packages that have been successfully installed in that environment.

Analysis of Other Options
  • Available commands are shown by pip help.
  • Outdated packages are shown by pip list --outdated.
  • All packages at PyPI are searched using pip search or a web browser.

Confirmation: pip list is used to view the list of installed packages in the local environment.

Question 15: PIP Search Command (Select two)

What is true about the pip search command?

Explanation

pip search queries the centralized **Python Package Index (PyPI)** database for available packages, which is an external online service, hence it **requires an internet connection**.

Analysis of Other Options
  • It searches names, summaries, and keywords, not just names.
  • It searches PyPI (external), not locally installed packages.

Confirmation: The command queries the online PyPI index, making both statements true.

Question 16: PIP Install Command (Select two)

What is true about the pip install command?

Explanation

You can specify a version using package==1.2.3. When not in a virtual environment, the --user flag is necessary to install packages into the current user's local directory instead of the system-wide Python directory.

Analysis of Other Options
  • The version **can** be changed/specified.
  • --system is not a standard option; system-wide installation is often the default without a virtual environment or --user flag.

Confirmation: pip install supports version specification and the --user flag for local installation.

Question 17: PIP Update Command

What is true about updating already installed Python packages?

Explanation

The correct and efficient way to update a package is to reuse the pip install command along with the -U (or --upgrade) option: pip install -U <package_name>.

Analysis of Other Options
  • Uninstalling/reinstalling is unnecessarily complex.
  • Updates are not automatic.
  • There is no dedicated reinstall command for updating (it's --force-reinstall for reinstalling).

Confirmation: Updating is handled by pip install --upgrade (or -u).

Question 18: PIP Removal Command

Which command can you use pip to remove an installed package?

Explanation

The dedicated subcommand for removing installed Python packages from the environment is uninstall: pip uninstall <package_name>.

Analysis of Other Options
  • remove is not a valid pip subcommand.
  • The other options use incorrect syntax or inappropriate subcommands (install).

Confirmation: The command for package removal is the explicit pip uninstall.
