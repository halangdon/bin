Module 3 — The Interaction Design Process

Overview

Interaction design is iterative: establish requirements, design alternatives, prototype, and evaluate. Repeat until the design meets user needs.

3.1 Iterative Cycle

Four core activities: Establish requirements, Design alternatives, Prototyping, Evaluating. Iterate frequently.

3.2 Establishing User Requirements

Methods: Stakeholder interviews, user observation, surveys, personas, scenarios. Output: user needs & functional/non-functional requirements.

3.3 Designing Alternatives

Brainstorm, sketch, storyboard, parallel design. Diverge before you converge.

3.4 Prototyping

Lo-Fi (paper) vs Hi-Fi (interactive). Use Lo-Fi early to test flows; Hi-Fi later to test visuals and micro-interactions.

3.5 Evaluating

Heuristic evaluation, usability testing, think-aloud protocol. Evaluation informs the next iteration.
