Module 4 — Establishing User Requirements

Overview

This module covers how to gather and specify user requirements — the foundation for user-centered design.

Definition

User requirements describe what a system should do, who will use it, and under what conditions — focusing on user goals, not just features.

Methods for Gathering Requirements

Stakeholder interviews, user observation, surveys/questionnaires, personas, scenarios.

Outcomes

Clear user needs, functional requirements (what system must do), and non-functional requirements (speed, accessibility, security).

Key Terms
  • Persona — fictional profile of a typical user
  • Scenario — story of how a persona uses the system
